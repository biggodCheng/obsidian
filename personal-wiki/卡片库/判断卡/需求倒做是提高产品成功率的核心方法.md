# 需求倒做是提高产品成功率的核心方法

## 判断内容

"需求倒做"是把产品开发的逻辑完全倒过来的方法论：先找到核心需求点，再设计产品，再开发产品，然后找到与之匹配的独特渠道。

## 核心流程

```
核心需求点 → 产品设计 → 产品开发 → 独特渠道
```

## 与供给驱动的对比

| 需求倒做 | 供给驱动 |
|---------|---------|
| 先找需求，再做产品 | 发现技术/Prompt，拍脑门套壳 |
| 从用户问题出发 | 从技术能力出发 |
| 找独特渠道匹配产品 | 产品做出来再想办法搞流量 |

## 为什么有效

- 确保解决的是真实需求，而非伪需求
- 避免做出"平平无奇、不创造增量价值"的产品
- 产品和渠道是匹配设计的，获客更容易

## 适用条件

- 任何新产品的开发
- 独立开发者寻找产品方向
- B2C产品尤其适用

## 不适用条件

- 纯技术探索项目
- 学术研究项目

## 验证案例

- 杜国楹连续打造背背佳、好记星、E人E本、8848手机、小罐茶等爆款，成功率100%
- "大模型六小虎"从技术供给出发，今天几乎都遇到瓶颈

## 口诀

**先有用户，再做产品**
