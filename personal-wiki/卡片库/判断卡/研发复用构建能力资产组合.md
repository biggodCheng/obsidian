# 研发复用构建能力资产组合

## 判断内容

**他不是在跑六个产品，他是在经营一个"能力资产组合"。**

zwibbler和rapt.ink共享同一套矢量引擎——一个卖给公司嵌入，一个卖给用户直接用。同一套canvas代码，两种商业模式。

Go + SQLite + 自写auth——六个产品全在用。

**判断标准**：
- 如果每个产品都从零开发 → 不是在经营能力资产
- 如果多个产品共享核心能力 → 在经营能力资产组合
- 如果你的能力可以复用到多个场景 → 有资产组合潜力

**研发复用的优势**：

**1. 效率倍增**：
- 同一套代码，多种商业模式
- 新产品开发时间大幅缩短
- 维护成本分摊到多个产品

**2. 风险分散**：
- 不是依赖单一产品
- 多个收入来源
- 一个产品失败不影响整体

**3. 能力沉淀**：
- 核心能力越来越强
- 进入壁垒越来越高
- 时间越久优势越大

**Steve的能力资产**：
- 矢量引擎（支撑zwibbler和rapt.ink）
- Go + SQLite + 自写auth（支撑所有六个产品）
- 本地AI推理工具链（支撑股票分析产品）

**适用场景**：
- 有多个产品想法的独立开发者
- 想要提升研发效率的团队
- 有技术能力想产品化的开发者

**不适用场景**：
- 单一产品专注策略
- 产品之间没有技术关联性
- 快速试错需要频繁换技术栈

**核心原则**：
不是在跑六个产品，是在经营一个"能力资产组合"。
