# 两天内做MVP的方法

## 前置假设

- 选择海外网站作为产品形态
- 提前想好需要验证的内容
- 只写一个主函数

## 分步流程

### 核心原则

作为一个海外工具网站产品的MVP，只有两点重要：
1. 界面要漂亮，看上去就很贵很专业
2. 核心的功能，要可用

### 时间分配

| 步骤 | 时间 | 说明 |
|------|------|------|
| 买模板 | 1小时 | 推荐Tailwind UI、Themeforest |
| 买域名 | 1小时 |  |
| 写文案 | 1小时 |  |
| 改模板 | 2小时 |  |
| 主函数 | 8小时 | 往往更少 |
| 部署上线 | 2小时 |  |
| **总计** | **15小时** | **两天内完成** |

### 界面如何漂亮

**花钱买个模板！**

推荐：
- Tailwind UI官方模板（如Salient）
- Themeforest.net（选择多，10-50美元）

### 核心功能如何完成

**只写一个函数！**

对，只写一个，这个函数就是你的主功能。其他函数一概不写。

任何一个函数，我不相信一个合格的程序员、在开源社区的帮助下，8个小时内写不出来。

**示例**：下载YouTube视频工具
- 输入：URL
- 输出：视频文件
- 去Github找代码，稍微改一改，全程不超过1小时

### 上线后

把链接发到一些Startup社区，比如Product Hunt。然后静静地等待数据。

### 观察数据

如果是免费产品，请重点关注**自然流量的增长**，因为它们代表用户在自传播。

自然流量包括：
- 社交网络流量
- 搜索流量
- 外链流量

## 预期产出

一个能够验证核心假设的MVP产品，部署上线，开始收集用户数据。

## 已知局限

- 需要基本的编程能力
- 需要购买模板和域名的预算
- 时间规划需要严格执行
