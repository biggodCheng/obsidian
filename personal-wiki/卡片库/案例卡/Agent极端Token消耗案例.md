# Agent极端Token消耗案例

## 案例内容
一个开发者分享的极端案例：49 个子任务并行跑了 2.5 小时，估计花费 8000 到 15000 美金。

## 关键数据
- 子任务数：49 个并行
- 运行时间：2.5 小时
- 预估成本：8000-15000 美金

## 案例分析
- Agent 模式下每轮都带完整上下文（滚雪球效应）
- 49 个子任务各自独立滚雪球，累积消耗巨大
- 即使单个任务看起来简单，内部可能转十几轮
- 每一轮都为前面所有上下文付费

## 经验教训
- 不理解 Token 就不理解账单
- Agent 模式必须配合优化手段（/clear、/compact、Subagent 隔离）
- 并行任务越多，总消耗可能远超预期
- 需要设置成本上限和监控
