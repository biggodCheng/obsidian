# ob-todo - 任务卡管理技能

## 定义

**龙虾AI的核心技能之一，提供工作卡片和任务管理能力，支持快速记录、任务分类和状态跟踪**

## 核心定位

- **角色**: 知识工作者的"任务秘书"
- **能力**: 快速捕捉任务，智能分类管理
- **价值**: 确保重要事项不遗漏，提升执行力

## 主要功能

### 1. 快速任务捕捉

支持自然语言快速记录任务

**示例**:
```
用户: "记一下明天下午3点开会"
AI: 自动创建任务卡片，设置时间、提醒

用户: "待办：跟进客户A的合同续签"
AI: 创建待办任务，关联客户信息
```

### 2. 智能任务分类

自动识别任务类型和优先级

**分类维度**:
- **紧急程度**: 紧急 / 不紧急
- **重要程度**: 重要 / 不重要
- **任务类型**: 会议 / 待办 / 提醒 / 跟进

### 3. 任务状态管理

跟踪任务状态和进度

**状态流转**:
```
待办 → 进行中 → 已完成 → 已归档
  ↓
被延期
  ↓
已取消
```

### 4. 提醒和通知

基于时间和事件的智能提醒

**提醒类型**:
- 时间提醒: "明天下午3点开会"
- 事件提醒: "客户A回复后提醒我"
- 循环提醒: "每周五下午5点总结本周工作"

## 任务卡片结构

### 元数据格式

```yaml
---
type: task
priority: high/medium/low
status: pending/in_progress/completed/cancelled
tags: [标签1, 标签2]
due_date: 2026-04-10
created: 2026-04-09
---
```

### 内容结构

- **任务描述**: 清晰的任务说明
- **背景信息**: 为什么需要这个任务
- **相关资源**: 相关的判断卡、方法卡
- **验收标准**: 如何判断任务完成
- **依赖关系**: 依赖的其他任务或资源

## 任务管理策略

### 1. 四象限法则

根据紧急度和重要性分类：

```
重要且紧急     重要不紧急
→ 立即处理      → 安排时间

紧急不重要     不重要不紧急
→ 快速处理      → 考虑删除
```

### 2. 时间块管理

将任务分配到具体时间段：

```
09:00-11:00  深度工作块
11:00-12:00  会议时间块
14:00-15:00  浅度工作块
15:00-16:00  任务处理块
```

### 3. 能量管理

根据个人能量水平安排任务：

```
高能量时段 → 重要且困难的任务
中能量时段 → 常规任务
低能量时段 → 琐碎任务
```

## 智能特性

### 1. 自动建议

基于历史数据提供建议：

```
AI: "根据历史数据，周五下午通常不适合安排重要会议，建议改到周一上午"
```

### 2. 冲突检测

检测时间冲突：

```
AI: "检测到冲突：已有一个会议安排在明天下午3点，是否需要调整？"
```

### 3. 依赖解析

解析任务依赖关系：

```
AI: "任务A依赖任务B完成，建议先完成B再开始A"
```

## 与其他技能的协作

```
ob-search (查找相关信息)
    ↓
ob-todo (创建行动计划)
    ↓
ob-note (记录执行过程)
    ↓
article-processor (总结经验)
```

**协作流程**:
1. 用 ob-search 查找相关经验和判断
2. 用 ob-todo 创建行动计划
3. 用 ob-note 记录执行过程和洞察
4. 用 article-processor 总结经验教训
