# ob-search - 全库检索技能

## 定义

**龙虾AI的核心技能之一，提供全库范围的精准检索能力，支持关键词搜索、标签过滤和卡片类型过滤**

## 核心定位

- **角色**: 知识库的"搜索引擎"
- **能力**: 快速定位相关卡片和信息
- **价值**: 从海量笔记中精准找到需要的内容

## 主要功能

### 1. 关键词搜索

基于关键词的全库搜索

**示例**:
```
用户: "找一下关于用户转化的判断卡"
AI: 返回所有包含"用户转化"的判断卡，按相关度排序
```

### 2. 标签过滤

根据标签筛选相关内容

**示例**:
```
用户: "找一下所有带[定价]标签的卡片"
AI: 返回所有标签包含"定价"的卡片
```

### 3. 卡片类型过滤

按卡片类型（判断卡、方法卡、案例卡、信息卡）过滤

**示例**:
```
用户: "找一下所有的判断卡"
AI: 返回所有类型为judgment的卡片
```

### 4. 组合查询

支持多条件组合查询

**示例**:
```
用户: "找一下带[产品]标签的判断卡和方法卡"
AI: 返回符合条件的相关卡片
```

## 技术特点

### 1. 深度内容理解

- 不仅匹配关键词，还理解语义
- 支持同义词和相关概念搜索
- 识别上下文关系

### 2. 智能排序

- 按相关度排序结果
- 考虑卡片类型（判断卡优先）
- 考虑信心程度（高信心优先）
- 考虑时间新鲜度（最新优先）

### 3. 结构化输出

- 返回匹配的卡片内容
- 高亮关键词匹配
- 提供上下文片段
- 列出相关卡片链接

## 应用场景

### 场景 A: 快速信息查找

```
用户: "XXX 在哪？"
AI: 找到包含"XXX"的所有卡片，列出位置
```

### 场景 B: 主题探索

```
用户: "关于定价我们有什么判断？"
AI: 返回所有与定价相关的判断卡
```

### 场景 C: 决策支持

```
用户: "我们之前对A/B测试有什么经验？"
AI: 搜索并返回所有A/B测试相关的卡片
```

### 场景 D: 知识回顾

```
用户: "我们总结过哪些提升转化的方法？"
AI: 返回所有转化相关的方法卡和案例卡
```

## 使用方式

### 自然语言交互

```
用户: "找一下关于用户留存的所有判断卡"
AI: [执行搜索，返回结果]
```

### 命令式调用

```
用户: "search: 留存率 标签:增长 类型:method"
AI: [执行结构化查询]
```

### 模糊查询

```
用户: "我们之前有没有讨论过类似的问题？"
AI: [基于语义理解进行搜索]
```

## 检索策略

### 1. 全文检索

- 搜索所有卡片的内容
- 包括标题、正文、标签
- 匹配关键词和语义

### 2. 结构化检索

- 按元数据字段检索
- 按卡片类型检索
- 按时间范围检索
- 按信心程度检索

### 3. 关联检索

- 基于wikilinks发现关联卡片
- 搜索相关概念的卡片
- 扩展到同义词和相关词

## 输出格式

### 简洁模式

```markdown
找到 3 个相关卡片:

1. [[用户留存优化判断卡]] - 信心:高
2. [[留存率提升方法]] - 信心:中
3. [[某产品留存案例]] - 信心:高
```

### 详细模式

```markdown
## 搜索结果: "用户留存" (3个卡片)

### 1. [[用户留存优化判断卡]]
- **类型**: 判断卡
- **信心**: 高
- **标签**: [增长, 留存, 用户]
- **摘要**: ...
- **相关片段**: ...
- **关联卡片**: ...
```

## 最佳实践

### 1. 使用精准关键词

- ❌ "增长"（太宽泛）
- ✅ "B2B SaaS 用户留存率增长策略"（精准）

### 2. 利用标签过滤

- 先搜索大主题（如"定价"）
- 再用标签细化（如标签:"B2B"）

### 3. 组合卡片类型

- 搜索判断卡获取决策依据
- 搜索方法卡获取操作步骤
- 搜索案例卡获取验证事实

### 4. 关注相关卡片

- 查看搜索结果的"相关卡片"
- 可能找到意想不到的关联

## 与其他技能的协作

```
ob-search (检索)
    ↓
ob-note (记录)
    ↓
ob-todo (行动)
    ↓
article-processor (学习)
```

**协作流程**:
1. 用 ob-search 查找相关信息
2. 用 ob-note 记录新的洞察
3. 用 ob-todo 创建行动计划
4. 用 article-processor 学习新内容
