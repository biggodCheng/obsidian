# article-processor - 文章处理技能

## 定义

**龙虾AI的核心技能之一，提供研究和探索的自动化内容处理能力，支持URL内容提取、智能拆卡和知识整合**

## 核心定位

- **角色**: 知识工作者的"研究助理"
- **能力**: 自动处理文章内容，提炼核心洞察
- **价值**: 从海量内容中快速提取有价值的信息

## 主要功能

### 1. URL内容处理

直接处理URL链接的内容

**示例**:
```
用户: "帮我处理这个URL：https://example.com/article"
AI: 
- 下载文章内容
- 提取正文
- 识别关键信息
- 创建摘要卡片
```

### 2. 智能拆卡

从长文中自动提炼判断卡

**拆卡流程**:
```
长文内容
    ↓
识别关键段落
    ↓
提取判断、方法、案例
    ↓
创建结构化卡片
    ↓
关联已有知识
```

### 3. 内容分类

自动识别文章类型和主题

**文章类型**:
- **研究论文**: 学术文章、研究报告
- **行业文章**: 行业分析、趋势报告
- **案例研究**: 具体案例、成功故事
- **教程指南**: 操作教程、最佳实践

### 4. 知识整合

将新内容整合到现有知识网络

**整合方式**:
- 识别与已有卡片的关系
- 更新相关卡片的信心程度
- 建立新的连接和关联
- 标识需要探索的领域

## 处理流程

### 标准流程

```
输入 (URL/文件)
    ↓
1. 内容提取
    ↓
2. 结构分析
    ↓
3. 核心识别
    ↓
4. 卡片创建
    ↓
5. 知识整合
    ↓
输出 (摘要+卡片)
```

### 详细步骤

#### 步骤 1: 内容提取

- 下载URL内容或读取文件
- 提取正文，去除广告和噪声
- 识别标题、作者、发布时间

#### 步骤 2: 结构分析

- 分析文章结构
- 识别关键章节
- 提取核心论点

#### 步骤 3: 核心识别

- 识别判断（决策依据）
- 识别方法（具体做法）
- 识别案例（真实事例）
- 识别数据（支撑信息）

#### 步骤 4: 卡片创建

- 为每个核心洞察创建卡片
- 设置卡片类型和元数据
- 建立卡片间的关联

#### 步骤 5: 知识整合

- 关联已有知识卡片
- 更新相关信心程度
- 标识知识空白
- 建议进一步探索

## 内容类型处理

### 研究论文

**处理重点**:
- 提取研究问题和结论
- 识别研究方法和数据
- 提炼可应用的建议
- 标记研究局限性

**输出**:
- 研究摘要卡
- 方法论卡片
- 数据信息卡

### 行业报告

**处理重点**:
- 提取行业趋势
- 识别关键数据
- 标记竞争情报
- 提炼行业洞察

**输出**:
- 行业趋势判断卡
- 数据信息卡
- 竞争情报卡

### 案例研究

**处理重点**:
- 提取案例背景
- 识别行动方案
- 提炼结果数据
- 总结成功/失败因素

**输出**:
- 案例摘要卡
- 行动方法卡
- 结果验证卡

### 教程指南

**处理重点**:
- 提取操作步骤
- 识别工具和方法
- 标记注意事项
- 提炼最佳实践

**输出**:
- 方法卡片
- 操作检查清单
- 工具推荐卡

## 智能特性

### 1. 内容质量评估

```
AI: "这篇文章质量评分: 8/10
    内容原创，数据详实，有实践价值
    建议创建3个判断卡和1个方法卡"
```

### 2. 关键性识别

```
AI: "这篇文章提出了反直觉的观点：
    '价格敏感度在不同行业差异巨大'
    建议创建反直觉判断卡"
```

### 3. 矛盾标识

```
AI: "警告：这篇文章的观点与已有的XXX判断卡矛盾
    建议对比验证后再决定是否采纳"
```

### 4. 增量更新

```
AI: "这篇文章补充了已有的XXX方法卡
    建议更新方法卡，添加新的实践案例"
```
