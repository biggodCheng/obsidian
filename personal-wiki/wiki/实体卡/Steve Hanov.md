# Steve Hanov

## 基本信息

**身份**：加拿大 Waterloo 独立开发者

**成就**：一个人运营六个产品，每个 $10K+ MRR，月技术开销仅 $20

## 产品矩阵

| 产品 | 类型 | 商业模式 |
|------|------|----------|
| websequencediagrams.com | UML时序图工具 | Freemium + 企业版（2008年老兵） |
| zwibbler.com | 嵌入式白板SDK | B2B永久许可 $5,999/单 |
| rhymebrain.com | 8语种押韵词典 | 广告 + API |
| rapt.ink | 浏览器矢量图形编辑器 | C端Freemium |
| eh-trade.ca | 加美股智能分析 | 金融订阅 |
| xreplyextension.com | X/Twitter AI回复扩展 | 一次性点数 $9/$19/$49 |

## 技术选型

**核心栈**：Go + SQLite + 单机 VPS + 自写 auth

**月成本**：$20

**原则**：
- SQLite + WAL（比跨网络Postgres快10-40倍）
- Go静态二进制（部署革命，几MB单文件）
- 本地AI推理（RTX 3090 + VLLM，批量推理永久免费）

## 产品策略

**B2B养C端**：
- B2B大额订单给底气做C端免费产品
- 经济好时B2B好卖，经济差时C端好卖（抗周期）

**研发复用**：
- 同一套矢量引擎支持多个产品
- 不是在跑六个产品，是在经营一个"能力资产组合"

**慢做、深做、长做**：
- 每3-5年才加一个新产品
- 时间护城河：SEO权重和反链接无法追赶

**定价模式**：
- 六种定价匹配品类最自然的用户心理
- 一次性点数付费规避订阅疲劳

## 核心观点

- **把成本压到接近零，和拿到一百万美元融资，给你的生存时间是一样长的**
- **先设定约束，再从约束出发选技术**
- **技术开销不应该成为阻碍你去试的理由**
