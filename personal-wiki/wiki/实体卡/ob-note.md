# ob-note - 笔记记录技能

## 定义

**龙虾AI的核心技能之一，提供随时记录笔记和洞察的能力，支持快速捕捉、智能分类和自动关联**

## 核心定位

- **角色**: 知识工作者的"灵感捕捉器"
- **能力**: 快速记录想法，自动分类归档
- **价值**: 确保有价值的洞察不丢失

## 主要功能

### 1. 快速笔记记录

支持自然语言快速记录笔记

**示例**:
```
用户: "这个洞察很重要，记个笔记"
AI: 自动创建笔记卡片，提示添加类型和标签

用户: "记一下：用户在周五下午更容易接受营销邮件"
AI: 创建观察笔记，建议关联相关概念
```

### 2. 智能笔记分类

自动识别笔记类型和主题

**笔记类型**:
- **洞察笔记**: 有价值的想法和洞察
- **观察笔记**: 观察到的现象和模式
- **经验笔记**: 从经验中学到的教训
- **疑问笔记**: 需要进一步探索的问题
- **灵感笔记**: 突然的想法和创意

### 3. 自动关联

自动关联相关笔记和概念

**关联方式**:
- 识别关键词，关联相关卡片
- 基于wikilinks建立连接
- 建议补充缺失的关联

### 4. 洞察提炼

从原始记录中提炼核心洞察

**提炼过程**:
```
原始记录
    ↓
识别核心观点
    ↓
提炼判断或方法
    ↓
创建结构化卡片
```

## 笔记卡片结构

### 元数据格式

```yaml
---
type: note/insight/observation/question
confidence: high/medium/low
tags: [标签1, 标签2]
status: new/growing/mature
created: 2026-04-09
source: [来源，如会议、阅读、思考]
---
```

### 内容结构

- **核心观点**: 一句话概括
- **详细描述**: 完整的内容描述
- **背景信息**: 为什么有这个洞察
- **应用建议**: 如何应用这个洞察
- **关联内容**: 相关的判断卡、方法卡

## 智能特性

### 1. 自动完成

基于上下文自动补全内容：

```
AI: "检测到你正在记录定价相关内容，
    是否关联到已有的定价判断卡？"
```

### 2. 模式识别

识别笔记中的模式：

```
AI: "你多次提到'周五下午效果更好'，
    这可能是一个有价值的模式，
    是否创建模式判断卡？"
```

### 3. 矛盾检测

检测与已有知识的矛盾：

```
AI: "这个新观察与已有的判断卡矛盾，
    是否需要更新判断卡的信心程度？"
```

## 笔记类型详解

### 1. 洞察笔记 (Insight)

最有价值的笔记类型，记录有价值的想法和洞察

**示例**:
- "用户对价格不敏感，但对价值很敏感"
- "B2B客户的决策周期比预期长2倍"

### 2. 观察笔记 (Observation)

记录观察到的现象和模式

**示例**:
- "用户在周五下午更容易接受营销邮件"
- "竞品在月初通常会发布更新"

### 3. 经验笔记 (Lesson)

从经验中学到的教训

**示例**:
- "不与用户确认需求就直接开发是个坏主意"
- "过早优化是万恶之源"

### 4. 疑问笔记 (Question)

记录需要进一步探索的问题

**示例**:
- "为什么我们的转化率突然下降？"
- "竞品的这个策略为什么有效？"

## 与其他技能的协作

```
ob-search (查找相关信息)
    ↓
ob-note (记录学习过程)
    ↓
ob-todo (创建行动计划)
    ↓
article-processor (深度学习)
```

**协作流程**:
1. 用 ob-search 查找相关背景
2. 用 ob-note 记录学习过程和洞察
3. 用 ob-todo 创建行动计划
4. 用 article-processor 深度学习相关内容

## 输出格式

### 笔记卡片

```markdown
## 用户行为洞察

**类型**: 洞察笔记
**时间**: 2026-04-09 15:30
**来源**: 产品评审会议

### 核心观点
用户对价格不敏感，但对价值非常敏感

### 详细描述
在产品评审中发现，用户对10%的价格上涨没有太大反应，
但对产品价值的认知直接影响购买决策

### 应用建议
- 定价策略应该基于价值而非成本
- 营销重点应该放在价值传递上
- 可以适当提高价格以体现价值

### 关联内容
- [[定价策略判断卡]]
- [[用户心理洞察]]
- [[价值营销方法]]
```

## 最佳实践

### 1. 即时捕捉

```
✅ 好主意出现时立即记录
❌ 等有空再记录（通常会忘记）
```

### 2. 完整记录

```
✅ 记录完整的上下文和思考过程
❌ 只记录一句话，忘记背景
```

### 3. 定期回顾

```
✅ 每周回顾笔记，提炼洞察
❌ 记录后就再也不看
```

### 4. 主动提炼

```
✅ 从笔记中提炼判断卡和方法卡
❌ 笔记永远停留在原始状态
```

## 笔记进化机制

```
新笔记 (New)
  ↓ 被引用或应用1次
成长笔记 (Growing)
  ↓ 被引用或应用3次以上
成熟笔记 (Mature)
  ↓ 内容被提炼为判断卡/方法卡
已整合 (Integrated)
```
