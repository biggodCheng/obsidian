# 龙虾AI (Lobster AI)

## 定义

**一个基于 Claude Code 的 Obsidian 自动化工具集，作为知识管理的"自动化机械臂"**

## 核心定位

- **角色**: 动态的执行引擎
- **能力**: 直接读写底层 Markdown 文件
- **价值**: AI 需要「直接下场搬砖」的权限

## 四个核心技能

| 技能 | 用途 | 示例 |
|------|------|------|
| **ob-search** | 提供全库基础检索能力 | "找一下 xxx" / "xxx 在哪" |
| **ob-todo** | 工作卡片/任务管理 | "记一下明天开会" / "待办 xxx" |
| **ob-note** | 随时记录/笔记管理 | "这个洞察很重要，记个笔记" |
| **article-processor** | 研究探索/内容处理 | "帮我处理这个 URL" |

## 核心价值

### 1. 自动化执行

直接操作底层 Markdown 文件，无需复制粘贴

### 2. 智能提取

从长文中自动提炼判断卡

### 3. 关联发现

自动发现相关卡片

### 4. 工作流编排

协调多个操作步骤

## 应用场景

### ob-search - 全库检索

```
用户: "找一下关于用户转化的判断卡"
AI: 返回所有相关卡片，按相关度排序
```

### ob-todo - 任务管理

```
用户: "记一下明天下午3点开会"
AI: 自动创建日常工作片，设置提醒
```

### ob-note - 笔记记录

```
用户: "这个洞察很重要，记个笔记"
AI: 创建新笔记，提示添加类型和标签
```

### article-processor - 文章处理

```
用户: "帮我处理这个URL"
AI: 下载文章 → 提取正文 → AI拆卡 → 保存到库
```

## 技术特点

1. **本地优先**: 数据永远在本地，AI 直接操作文件系统
2. **深度集成**: 与 Obsidian 深度集成，可读写所有内容
3. **原子化操作**: 所有操作都是原子化的，可组合使用
4. **结构化输出**: 输出符合标准格式的 Markdown 文件
