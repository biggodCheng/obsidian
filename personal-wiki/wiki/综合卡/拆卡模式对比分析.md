# 拆卡模式对比分析 - Lobster vs LLM Wiki

## 摘要

本文对比分析两种知识摄取和拆解模式：**Lobster-article 拆卡模式**和**LLM Wiki 摄取模式**，从设计理念、操作流程、优缺点等维度进行深度对比。

---

## 设计理念对比

### Lobster-article: 资产沉淀导向

**核心理念**: 不只是存信息，更是存「判断」

```
外部信息 → AI拆卡 → 提炼判断 → 应用验证 → 市场反馈 → 优化迭代
     ↑                                                    │
     └────────────────────────────────────────────────────┘
```

**核心洞察**: 只有提取出顶层的【判断卡】，第二大脑才具备真正的商业护城河。

### LLM Wiki: 知识网络导向

**核心理念**: LLM 增量构建和维护持久化 wiki

```
┌─────────────────────────────────────────────────────────────┐
│                        Raw Sources                           │
│  (不可变 - 你的源文件，LLM 只读不写)                          │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                          The Wiki                            │
│  (LLM 拥有和维护 - 你读，LLM 写)                              │
│  摘要、实体页面、概念页面、对比、概览、综合                     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                          The Schema                          │
│  (配置文件 - 告诉 LLM 如何结构化和维护 wiki)                   │
└─────────────────────────────────────────────────────────────┘
```

**核心洞察**: Wiki 是一个持久的、复合的制品。交叉引用已经存在。矛盾已经被标记。

---

## 卡片/页面分类对比

### Lobster-article: 四层结构（按价值排序）

| 类型 | 价值 | 解决问题 | 示例 |
|------|------|----------|------|
| **判断卡** | ⭐⭐⭐⭐⭐ | 怎么判断 | 什么情况下用A而不是B |
| **方法卡** | ⭐⭐⭐⭐ | 具体做法 | 拆卡的标准流程(SOP) |
| **案例卡** | ⭐⭐⭐ | 真实发生的事 | 客户A的转化率提升过程 |
| **信息卡** | ⭐ | 是什么 | 产品参数 |

**价值判断**: 信息卡价值最低，建议谨慎创建。

### LLM Wiki: 页面类型分类

| 类型 | 存放位置 | 内容 | 示例 |
|------|----------|------|------|
| **实体** | 实体卡/ | 人名、组织、产品 | 袁锐钦、乔纳·伯杰 |
| **概念** | 概念卡/ | 理论、方法、术语 | 社交货币、病毒循环 |
| **摘要** | 摘要卡/ | 源文件摘要 | SBTI传播核武器拆解 |
| **综合** | 综合卡/ | 查询归档、分析报告 | 判断卡应用指南 |

**分类逻辑**: 按内容性质分类，而非价值排序。

## 元数据结构对比

### Lobster-article: 状态机驱动

```yaml
---
type: 判断卡/方法卡/案例卡/信息卡
confidence: high/medium/low
tags: [标签1, 标签2]
status: new/growing/mature
created: 2026-03-30
updated: 2026-03-30
---
```

**状态流转规则**:
- New → Growing: 被引用1次或应用于1个真实项目
- Growing → Mature: 被引用3次以上且反馈良好
- Mature → Outdated: 发现部分条件变化不再适用
- 任何状态 → Discarded: 被市场证伪

### LLM Wiki: 关系驱动

```yaml
---
title: 页面标题
type: 实体卡/概念卡/摘要/综合
date: YYYY-MM-DD
tags: [tag1, tag2]
sources: [source1, source2]
related: [page1, page2]
---
```

**关联机制**:
- `sources`: 指向原始源文件
- `related`: 指向相关页面（双向链接）

## 操作流程对比

### Lobster-article: 拆卡流程

```
输入 (URL/文件)
    ↓
1. 内容提取
    ↓
2. 结构分析
    ↓
3. 核心识别 (判断/方法/案例/数据)
    ↓
4. 卡片创建
    ↓
5. 知识整合
    ↓
输出 (摘要+卡片)
```

**重点**: 提炼判断卡（最高价值）

### LLM Wiki: 摄取流程

```
1. 读取源文件
    ↓
2. 与用户讨论关键要点
    ↓
3. 在 wiki 中写入摘要页面
    ↓
4. 更新整个 wiki 中的相关实体和概念页面
    ↓
5. 更新索引
    ↓
6. 在日志中追加条目
```

**重点**: 维护 wiki 的整体健康和一致性

## 优缺点对比

### Lobster-article 优势

| 优势 | 说明 |
|------|------|
| **价值导向** | 明确的价值排序（判断卡最高），避免低价值信息堆积 |
| **闭环系统** | 知识进化机制，每次应用都成为下一次输入 |
| **商业化** | 飞轮效应，市场反馈反哺知识优化 |
| **原子化** | 最小可复用单元，精准检索和组合 |
| **元数据驱动** | 信心程度、状态流转，可量化知识价值 |

### Lobster-article 劣势

| 劣势 | 说明 |
|------|------|
| **价值判断主观** | 什么值得判断卡？需要经验和判断 |
| **维护成本高** | 需要持续更新状态、验证知识 |
| **过度强调判断** | 可能忽视信息卡的基础价值 |
| **缺乏整体视图** | 卡片分散，难以把握全貌 |

### LLM Wiki 优势

| 优势 | 说明 |
|------|------|
| **结构清晰** | 三层架构（Raw/Wiki/Schema），职责分明 |
| **交叉引用** | 自动维护链接关系，发现隐藏连接 |
| **健康检查** | Lint机制，发现矛盾、孤立页面、缺失引用 |
| **查询归档** | 有价值的答案可作为新页面保存 |
| **持久化** | Wiki随时间增长，越来越丰富 |

### LLM Wiki 劣势

| 劣势 | 说明 |
|------|------|
| **缺乏价值排序** | 概念页面和实体页面没有价值区分 |
| **无闭环机制** | 知识不随应用和反馈进化 |
| **更新被动** | 依赖用户主动查询和摄取 |
| **过度复杂** | 对于简单拆卡场景可能过重 |

## 适用场景对比

### Lobster-article 更适合

1. **内容创作者**: 需要从知识中生产商业内容
2. **产品团队**: 需要沉淀判断和经验
3. **创业者**: 需要构建商业护城河
4. **研究人员**: 需要提炼核心洞察

### LLM Wiki 更适合

1. **知识管理**: 需要系统化组织知识
2. **学术研究**: 需要追踪概念和实体关系
3. **学习笔记**: 需要构建知识网络
4. **个人wiki**: 需要长期维护和查询

## 核心差异总结

| 维度 | Lobster-article | LLM Wiki |
|------|-----------------|----------|
| **目标** | 沉淀高价值判断 | 构建知识网络 |
| **价值导向** | 强（判断卡最高） | 无 |
| **闭环机制** | 有（市场反馈） | 无 |
| **原子化** | 强（最小单元） | 弱（页面较大） |
| **元数据** | 丰富（状态、信心） | 简单（标签、日期） |
| **维护** | 主动（应用驱动） | 被动（查询驱动） |
| **复杂度** | 中等 | 高 |
| **商业价值** | 高（飞轮效应） | 中（知识网络） |

## 融合建议

### 最佳实践：结合两者优势

1. **用 LLM Wiki 构建知识网络**
   - 实体页面追踪人名、组织、产品
   - 概念页面沉淀理论和方法
   - 摘要页面记录源文件

2. **用 Lobster-article 沉淀高价值判断**
   - 从文章中提炼判断卡
   - 建立知识进化机制
   - 构建商业闭环

3. **交叉引用建立连接**
   - 判断卡引用 wiki 中的概念
   - Wiki 页面引用判断卡作为案例

### 建议架构

```
┌─────────────────────────────────────────────────────────────┐
│                        Raw Sources                           │
│  源文件（不可变）                                              │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                    LLM Wiki (知识网络)                       │
│  实体 + 概念 + 摘要                                            │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│              Lobster Cards (价值资产)                        │
│  判断卡 + 方法卡 + 案例卡 + 信息卡                              │
└─────────────────────────────────────────────────────────────┘
                              ↓
                      应用验证 + 市场反馈
                              ↓
                      优化迭代（闭环）
```
