## 什么是判断卡？

### 定义

**判断卡 (Judgments) = 笔记四层结构中价值最高的卡片类型，用于记录"怎么判断"的决策依据**

### 核心价值

| 维度 | 说明 |
|------|------|
| **解决问题** | 「怎么判断」 |
| **典型示例** | 什么情况下用 A 而不是 B |
| **核心价值** | 高杠杆的核心资产，商业护城河 |

### 在笔记体系中的位置

```
价值金字塔 (从高到低)
┌─────────────────────────┐
│   判断卡 (Judgments)     │ ⭐⭐⭐⭐⭐ 最核心
├─────────────────────────┤
│   方法卡 (Methods)       │ ⭐⭐⭐⭐
├─────────────────────────┤
│   案例卡 (Cases)         │ ⭐⭐⭐
├─────────────────────────┤
│   信息卡 (Information)   │ ⭐ 最低价值
└─────────────────────────┘
```

**核心结论**: 只有提取出顶层的【判断卡】，你的第二大脑才具备真正的商业护城河。

## 如何创建判断卡？

### 第一步：识别判断机会

从日常工作和学习中，识别出以下类型的判断：

| 洞见类型 | 说明 | 示例 |
|----------|------|------|
| **观察 (Observation)** | 发现了什么现象 | 用户用触屏异常变长 |
| **模式 (Pattern)** | 识别出什么规律 | 降价10%增加30%销量 |
| **方法 (Method)** | 总结出什么做法 | A/B测试五步法 |
| **反直觉 (Counter-intuitive)** | 颠覆常识的发现 | 干关不是最佳节能状态 |

### 第二步：标准化格式

每张判断卡应包含以下结构：

#### 元数据 (Frontmatter)

```yaml
---
type: judgment
confidence: high/medium/low
tags: [标签1, 标签2]
status: new/growing/mature
created: 2026-04-09
updated: 2026-04-09
---
```

#### 内容结构

1. **场景描述**: 在什么情况下需要这个判断
2. **判断依据**: 为什么这样判断
3. **适用条件**: 什么情况下适用，什么情况下不适用
4. **验证次数**: 这个判断被验证过多少次

### 第三步：标注信心程度

| 等级 | 触发条件 | 示例 |
|------|----------|------|
| **高** | 30次以上验证 | 经多个项目反复验证的方法 |
| **中** | 1-2次随机验证 | 有一定依据但样本有限 |
| **低** | 未验证的想法 | 初步假设，待验证 |

### 实践示例

**场景**: 产品功能优先级判断

```markdown
---
type: judgment
confidence: medium
tags: [产品管理, 优先级, 决策]
status: growing
created: 2026-04-09
---

# 何时优先开发 AI 功能而非基础功能

## 场景描述
在资源有限的情况下，需要在 AI 创新功能和基础功能优化之间做选择。

## 判断依据
- 如果目标用户是早期采用者 → 优先 AI 功能
- 如果目标用户是主流用户 → 优先基础功能
- 如果竞争对手已有类似功能 → 必须跟进

## 适用条件
✅ 适用于: B2B SaaS 产品，技术敏感型用户
❌ 不适用于: 消费级 app，大众市场

## 验证记录
- 2026-03-15: 项目A验证成功 (用户增长+35%)
- 2026-04-01: 项目B验证失败 (用户无感知)
```

## 如何应用判断卡？

### 1. 建立知识原子化习惯

**原则**: 一卡一洞见，每张卡片只讲一个核心观点

| 传统笔记 | 原子化判断卡 |
|----------|-------------|
| 一篇3000字的读书笔记 | 拆分为15-20张独立判断卡 |
| 每张卡片混合多个观点 | 每张卡片只讲一个核心判断 |
| 难以检索复用 | 可精准检索组合 |

### 2. 触发知识进化机制

```
新生 (New)
  ↓ 被引用1次或应用于1个真实项目
成长 (Growing)
  ↓ 被引用3次以上且反馈良好
成熟 (Mature)
  ↓ 发现部分条件变化
过时 (Outdated)
  ↓ 被市场证伪
废弃 (Discarded)
```

**状态流转规则**:
- **New → Growing**: 被引用1次或应用于1个真实项目
- **Growing → Mature**: 被引用3次以上且反馈良好
- **Mature → Outdated**: 发现部分条件变化不再适用
- **任何状态 → Discarded**: 被市场证伪

### 3. 构建闭环系统

```
信息输入 → 提炼判断 → 应用验证 → 市场反馈 → 优化迭代
     ↑                                              │
     └──────────────────────┘
```

**实践步骤**:
1. **记录反馈**: 每次应用都记录市场反馈
2. **关联判断**: 将反馈关联到具体的判断卡
3. **更新信心**: 根据反馈更新信心程度
4. **优化内容**: 用优化后的判断生成新内容
5. **循环迭代**: 持续循环，知识越用越准

### 4. 实际应用场景

#### 场景 A: 快速决策

```
用户: "这个功能应该用 A 方案还是 B 方案？"
AI: 检索判断卡 → 返回相关判断 → 提供决策依据
```

#### 场景 B: 内容创作

```
创作需求: 写一篇关于产品定价的文章
步骤:
1. 检索"定价"相关判断卡
2. 组合多个判断卡形成观点
3. 用判断卡支撑论点
4. 发布后收集反馈
5. 更新判断卡的信心程度
```

#### 场景 C: 问题解决

```
遇到问题: 用户留存率下降
步骤:
1. 检索"留存率"相关判断卡
2. 找到历史验证过的提升方法
3. 应用到当前问题
4. 记录效果反馈
5. 更新判断卡状态
```

## 核心洞察

### 关键要点

1. **优先提取判断卡**: 从所有内容中优先提取判断卡，而非信息卡
2. **独立成卡**: 每个判断独立成一张卡片，便于检索和复用
3. **标注信心**: 明确标注信心程度，知道哪些判断更可靠
4. **持续验证**: 在实践中验证并更新状态，知识越用越准
5. **定期回顾**: 定期回顾低信心判断卡，寻找验证机会

### 终局洞察

**真正的基础设施不是帮你「记笔记」，而是构建一个闭环，让每一次市场反馈都成为下一次爆款的弹药。**
