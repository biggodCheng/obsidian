# Token经济学

## 摘要
Token 的计费逻辑：输出比输入贵 5 倍，隐性 Token 是最大盲区，缓存可省 90%。

## 输入 vs 输出价格（每百万 Token）
- Haiku：输入 $1 / 输出 $5
- Sonnet：输入 $3 / 输出 $15
- Opus：输入 $5 / 输出 $25

## 价格差异原因
- 输入 Token 可并行处理（一次性全看完）
- 输出 Token 必须逐个生成（串行过程，基于前面所有内容推算下一个字）
- 输出计算量远大于输入

## 隐性 Token（大多数人忽略）
- 系统提示词：每次对话都带，通常几千 Token
- 对话历史：20 轮聊天全在
- 工具描述：搜索、代码执行等工具定义
- 项目文件：AI 编程工具中打开的文件、项目配置
- 一条简单消息可能附带几万 Token 的「隐形行李」

## 核心原则
- 让 AI 少说废话，比少打字更省钱
- 上下文窗口是「视野」不是「记忆」
- 上下文越长，单次请求计算量越大
