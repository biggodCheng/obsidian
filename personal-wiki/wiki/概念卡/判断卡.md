# 判断卡 (Judgments)

## 定义

**笔记四层结构中价值最高的卡片类型，用于记录"怎么判断"的决策依据**

## 核心价值

- **解决**: 「怎么判断」
- **举例**: 什么情况下用 A 而不是 B
- **价值**: 高杠杆的核心资产，商业护城河

## 与其他卡片类型的对比

| 类型 | 解决问题 | 价值等级 | 示例 |
|------|----------|----------|------|
| **判断卡** | 怎么判断 | ⭐⭐⭐⭐⭐ | 什么情况下用 A 而不是 B |
| **方法卡** | 具体做法 | ⭐⭐⭐⭐ | 拆卡的标准流程 (SOP) |
| **案例卡** | 真实发生的事 | ⭐⭐⭐ | 客户 A 的转化率提升过程 |
| **信息卡** | 是什么 | ⭐ | 产品参数 |

## 核心结论

**只有提取出顶层的【判断卡】，你的第二大脑才具备真正的商业护城河。**

## 判断卡的要素

### 1. 元数据结构

```yaml
---
type: 判断卡
confidence: high/medium/low
tags: [标签1, 标签2]
status: new/growing/mature
created: 2026-04-09
updated: 2026-04-09
---
```

### 2. 内容结构

- **场景描述**: 在什么情况下需要这个判断
- **判断依据**: 为什么这样判断
- **适用条件**: 什么情况下适用，什么情况下不适用
- **验证次数**: 这个判断被验证过多少次

## 信心程度校准

| 等级 | 触发条件 | 示例 |
|------|----------|------|
| **高** | 30次以上验证 | 经多个项目反复验证的方法 |
| **中** | 1-2次随机验证 | 有一定依据但样本有限 |
| **低** | 未验证的想法 | 初步假设，待验证 |

## 状态流转规则

- **New → Growing**: 被引用1次或应用于1个真实项目
- **Growing → Mature**: 被引用3次以上且反馈良好
- **Mature → Outdated**: 发现部分条件变化不再适用
- **任何状态 → Discarded**: 被市场证伪

## 实践建议

1. **优先提取**: 从所有内容中优先提取判断卡
2. **独立成卡**: 每个判断独立成一张卡片
3. **标注信心**: 明确标注信心程度
4. **持续验证**: 在实践中验证并更新状态
5. **定期回顾**: 定期回顾低信心判断卡

## 分类边界：如何区分判断卡 vs 方法卡 vs 案例卡

### 判别树

```
这段内容回答的是什么类型的问题？

├─ "应该怎么判断/选择/决策？" → 判断卡
│   特征：有明确观点 + 因果推理 + 适用边界
│   测试：能否写成 "在 X 条件下，应该 Y，因为 Z"？
│
├─ "具体怎么做/按什么步骤？" → 方法卡
│   特征：有可执行的步骤、流程、检查清单
│   测试：读者看完能否直接照着做？
│
├─ "发生了什么事/有什么数据？" → 案例卡
│   特征：具体的时间、地点、人物、结果
│   测试：能否验证/证伪某个判断或方法？
│
└─ "是什么/有什么事实？" → 信息卡
```

### 最容易混淆的场景

| 混淆场景 | 正确分类 | 判断依据 |
|---|---|---|
| "A 的成功说明 B 逻辑是对的" | **判断卡** | 提炼的是因果判断，有观点 |
| "分析 A 产品为什么成功" | **判断卡** | 有因果推理 |
| "从 A 的做法中提炼出 B 方法" | **方法卡** | 提炼的是通用流程，可复用 |
| "如何复制 A 的成功" | **方法卡** | 有可执行步骤 |
| "A 产品用 B 方法取得了 C 效果" | **案例卡** | 有具体对象+具体结果，可验证 |
