# 最穷技术栈与独立开发

## 摘要

文章核心案例：加拿大独立开发者 [[Steve Hanov]]，一个人运营六个产品（每个 $10K+ MRR），月技术开销仅 **$20**。

**核心洞察**：把成本压到接近零，和拿到一百万美元融资，给你的生存时间是一样长的。

**技术选型**：Go + SQLite + 单机 VPS + 自写 auth

**六个产品**：websequencediagrams.com（UML时序图）、zwibbler.com（嵌入式白板SDK）、rhymebrain.com（押韵词典）、rapt.ink（矢量图形编辑器）、eh-trade.ca（股票分析）、xreplyextension.com（AI回复扩展）

## 四个反共识判断

1. **SQLite + WAL 就是你的 Postgres，而且快10倍**：进程内调用（纳秒级）vs跨网络调用（毫秒级），差距40倍
2. **Go的静态二进制是部署革命**：几MB单文件，无依赖，一台$5 VPS能撑每秒几万请求
3. **不要买"企业云思维"的账**：在用户都没看过landing page之前就启动EKS和RDS，六个"万一"加起来$3000
4. **家里的显卡就是无限AI额度**：$900一次性投入，批量推理永久免费

## 产品组合策略

### 洞察一：B2B养C端（抗周期组合）
- zwibbler一单$5,999，一年20单就是$120K ARR
- 大额B2B订单给底气做C端免费产品
- 经济好时B2B好卖，经济差时C端好卖

### 洞察二：研发复用
- zwibbler和rapt.ink共享同一套矢量引擎
- 同一套canvas代码，两种商业模式
- Go + SQLite + 自写auth，六个产品全在用
- 不是在跑六个产品，是在经营一个"能力资产组合"

### 洞察三：慢做、深做、长做
- 每3-5年才加一个新产品
- websequencediagrams从2008年做到现在，SEO权重和反链接无法追赶
- 这叫时间护城河

### 洞察四：定价模式比定价数字更重要
- 六种定价：永久许可、订阅、广告+API、Freemium、一次性点数
- 每一种都匹配品类最自然的用户心理

## 技术选型原则

**先设定约束，再从约束出发选技术。**

没有放之四海而皆准的"最佳技术栈"。

约束决定选择，而不是技术决定约束。

Steve的约束：一个人、$20/月、六个产品。

在这个约束下，Go + SQLite + 单机VPS几乎是唯一最优解。

## 生存公式

你能活多久 = 你有多少钱 ÷ 你每月花多少钱

**分子你控制不了**——VC不理你、市场不买账、产品没起来

**分母你百分百能控制**

把分母压到$20，你就可以一直活着，一直试，直到试对为止。

你就拥有了全世界最强的创业者状态：**不需要任何人的钱，也不需要任何人的同意。**
