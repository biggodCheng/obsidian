# Lobster Note - 笔记管理

快速创建 Obsidian 知识库中的笔记卡片。

## 执行入口

```
python .claude/skills/ob-note/note.py <类型> "<标题>" [--content "内容"] [--tags "标签1,标签2"] [--confidence low|medium|high]
```

依赖：`.lobster/lobster_utils.py`（核心工具模块）

## 工作流

### Step 1: 解析用户意图

从用户输入中提取以下信息：

| 信息 | 来源 | 缺失时处理 |
|------|------|-----------|
| 卡片类型 | 用户明确说"判断卡/方法卡/案例卡/信息卡" | **询问用户**选择哪种类型 |
| 标题 | 用户语句中的核心主张/方法名/事件名 | AI 从上下文提炼，**展示给用户确认** |
| 内容 | 用户描述的详细内容 | 使用模板默认内容 |
| 标签 | 从内容中提取关键词 | 至少生成 1 个标签 |
| 信心程度 | 用户未指定时默认 low | 可询问 |

**意图解析规则**：
- "记一下/这个洞察很重要" → 需要进一步询问类型
- "判断卡: xxx" → 类型=判断卡，标题=xxx
- "方法: xxx" → 类型=方法卡，标题=xxx
- 模糊输入（如"帮我创建一张卡片"）→ **必须询问**卡片类型

### Step 2: 展示预览并确认

创建前展示：

```
即将创建：
- 类型：判断卡
- 标题：降价10%可以增加30%销量
- 标签：[定价, 增长]
- 信心程度：low

确认创建？(Y/n)
```

用户确认后才执行 Step 3。

### Step 3: 执行创建

```bash
python .claude/skills/ob-note/note.py "判断卡" "降价10%可以增加30%销量" --content "..." --tags "定价,增长" --confidence low
```

### Step 4: 返回结果

返回创建的笔记信息：
- 文件路径
- 卡片类型
- 标题
- 标签

## 卡片类型速查

| 类型 | 回答的问题 | 何时用 |
|------|-----------|-------|
| **判断卡** | "该怎么判断？" | 有主张、可辩论、有适用条件 |
| **方法卡** | "具体怎么做？" | 有步骤、可复用的流程 |
| **案例卡** | "发生了什么？" | 真实事件、有因果和教训 |
| **信息卡** | "是什么？" | 难搜索+高频引用+战略重要（默认不建） |

## 各类型 Frontmatter 结构

**判断卡**：`type: 判断卡` + 主张 + 适用条件 + 不适用条件 + `confidence: low|medium|high`

**方法卡**：`type: 方法卡` + 前置假设 + 分步流程 + 预期产出

**案例卡**：`type: 案例卡` + 具体情境 + 发生了什么 + 为什么重要

**信息卡**：`type: 信息卡`（谨慎创建，能搜到就不建）

## 边界条件

- **目标文件已存在** → 询问用户：覆盖 / 新建（加后缀）
- **note.py 执行失败** → 检查 lobster_utils.py 是否存在，报错给用户
- **标题含特殊字符**（冒号、斜杠）→ 自动替换为中文标点或下划线
- **用户未指定内容** → 尝试使用 `templates/{类型}.md` 模板

## 使用示例

```
记一下判断卡: 降价10%可以增加30%销量
→ python note.py "判断卡" "降价10%可以增加30%销量" --tags "定价,增长"

创建方法卡: A/B测试五步法
→ python note.py "方法卡" "A/B测试五步法" --tags "测试,方法论" --confidence medium

这个洞察很重要：跨部门沟通要先对齐目标
→ 类型=判断卡, 标题=跨部门沟通对齐法则, 先确认再创建

帮我创建一张卡片
→ 询问：您想创建什么类型的卡片？（判断卡/方法卡/案例卡/信息卡）
```
